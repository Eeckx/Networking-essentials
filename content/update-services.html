🧩 Windows Server Update Services (WSUS) – Installatie, Configuratie en Beheer

🎯 Doelstelling

In dit lab leer je hoe je Windows Server Update Services (WSUS) installeert en configureert. WSUS stelt beheerders in staat om Microsoft-updates lokaal te beheren, goed te keuren en te distribueren binnen het netwerk, zodat niet elke client afzonderlijk updates hoeft te downloaden van het internet.

🧰 Voorwaarden

  • Windows Server 2019, 2022 of nieuwer
  • Statische IP-adres en correcte DNS-instelling
  • Lid van het domein (optioneel maar aanbevolen)
  • Voldoende schijfruimte voor updatebestanden (minstens 30–100 GB)
  • Internettoegang of verbinding met een upstream WSUS-server

⚙️ Installatie van WSUS

  1. Open Server Manager → Add Roles and Features.
  2. Kies Role-based or feature-based installation.
  3. Selecteer de juiste server.
  4. Vink Windows Server Update Services aan.
  5. Bij features, klik Next.
  6. Kies waar je updates wil opslaan:
    • Bijv. D:\WSUS\Updates (aanbevolen aparte schijf)
  7. Selecteer databaseoptie:
    • WID (Windows Internal Database) voor kleine/middelgrote omgevingen.
    • SQL Server als je al een SQL-instantie hebt.
  8. Klik Install.
  9. PowerShell alternatief: 
    
Install-WindowsFeature -Name UpdateServices,UpdateServices-WidDB,UpdateServices-Services -IncludeManagementTools

🔧 Eerste configuratie (Post-installatie)

  1. Open Server Manager → Klik op de waarschuwing “Post-deployment Configuration”.
  2. Klik op Launch Post-Installation Tasks om WSUS te initialiseren.
  3. Daarna open je de WSUS Management Console via Tools → Windows Server Update Services.
  4. De configuratiewizard start automatisch.

⚙️ Configuratiewizard doorlopen

  1. Kies of deze WSUS-server synchroniseert met:
    • Microsoft Update (direct van internet)
    • Upstream WSUS-server (intern mirror-systeem)
  2. Selecteer proxyserver indien nodig.
  3. Voer een eerste synchronisatie uit om producten en classificaties te laden.
  4. Kies welke producten je wilt beheren, bv.:
    • Windows 10, Windows 11, Windows Server 2022
    • Microsoft Office-producten
  5. Kies updateclassificaties:
    • Critical Updates
    • Security Updates
    • Service Packs
    • Feature Packs
  6. Kies synchronisatieplanning (dagelijks aanbevolen).
  7. Klik Finish en laat WSUS synchroniseren.

📦 Updategroepen en goedkeuringen

Na synchronisatie kun je updates goedkeuren en clients beheren.

  1. Maak computerdoelgroepen aan:
    • Servers
    • Workstations
    • Testgroep
  2. Rechtsklik op “All Computers” → Add Computer Group.
  3. Om updates goed te keuren:
    • Ga naar Updates → filter op “Needed Updates”.
    • Selecteer updates → Approve → kies doelgroep (bijv. Testgroep).
  4. Je kunt ook automatische goedkeuring instellen via:
    • Options → Automatic Approvals
    • Bepaal regels per classificatie of product.

💻 Clients koppelen via Group Policy

Gebruik Group Policy om clients te laten updaten via WSUS.

  1. Open Group Policy Management (gpmc.msc).
  2. Maak of bewerk een GPO, bv. “WSUS Settings”.
  3. Navigeer naar:
    • Computer Configuration → Policies → Administrative Templates → Windows Components → Windows Update → Specify intranet Microsoft update service location
  4. Zet op Enabled en vul in:
    • Intranet update service: http://wsusserver.lab.local
    • Intranet statistics server: http://wsusserver.lab.local
  5. Configureer eventueel:
    • Automatic Updates = Enabled
    • Schedule Install Day/Time = bv. elke nacht om 3u
  6. Voer op clients uit: 
    
gpupdate /force
wuauclt /detectnow
wuauclt /reportnow
  7. Controleer in WSUS-console of clients verschijnen onder “Computers”.
